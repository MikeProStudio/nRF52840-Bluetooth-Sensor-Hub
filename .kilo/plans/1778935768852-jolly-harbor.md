# Plan: OLED Display Redesign

## Problem

Current connected view (`oled_update_connected`, line 1049 in `src/main.c`) uses `oled_puts_w(..., 6)` — rendering the 8x8 font at only 6 pixels wide. This produces cramped, barely-legible text. All sensor info (accel, gyro, battery, power) is crammed into 4 lines of tiny text.

## Library Assessment

**No new library needed.** The SSD1306 is already driven optimally via Zephyr's `display` API + manual framebuffer. Alternatives considered:

| Library | RAM Cost | Flash Cost | Verdict |
|---------|----------|------------|---------|
| **Current (manual FB)** | 1KB framebuffer | minimal | ✅ Best for this use case |
| LVGL | 10KB+ RAM, widget trees | 100KB+ flash | ❌ Overkill for 128x64 monochrome |
| u8g2 | 1KB buffer, but full C lib | 20-50KB flash | ❌ Redundant — Zephyr already has display API |
| Adafruit GFX | 1KB buffer + ~4KB code | ~8KB flash | ❌ Needs port to Zephyr, no benefit over current |

Only 2KB of stack is allocated for the OLED workqueue — not enough for a heavy library. The nRF52840 has 256KB RAM so LVGL could technically run, but it adds unnecessary complexity and power draw.

**Recommendation:** Keep the manual framebuffer approach. Just fix the font width and redesign the layout.

## Proposal: 2-Page Auto-Scroll Layout

### Page 1 — "IMU Dashboard" (4s dwell)

```
┌──────────────────────────────────┐
│Accel X -0.02 m/s²                │  8x16 font, y=0
│Accel Y +0.01 m/s²                │  8x16 font, y=16
│Accel Z +9.81 m/s²                │  8x16 font, y=32
│Gyro  X +0.01 °/s                 │  8x16 font, y=48
└──────────────────────────────────┘
```

### Page 2 — "Power Status" (3s dwell)

```
┌──────────────────────────────────┐
│Gyro  Y +0.00 °/s                 │  8x16 font, y=0
│Gyro  Z +0.00 °/s                 │  8x16 font, y=16
│Bat  4123mV   89%                 │  8x16 font, y=32
│PWR  USB   TX +8dBm              │  8x16 font, y=48
└──────────────────────────────────┘
```

**Total info per page: 4 lines × 8x16 font.** Each line fits ~13-16 readable characters (8px/char × 16 chars = 128px).

## Implementation Changes (only `src/main.c`)

### 1. Replace `oled_update_connected()` with a 2-page function

Remove the old `oled_update_connected()` (lines 1049-1086) and add:

- `oled_draw_page_imu()` — renders accel/gyro values from `accel_data` / `gyro_data` using `oled_puts_8x16()` (full 8px wide chars)
- `oled_draw_page_power()` — renders gyro (remaining axes) + battery (`battery_status.voltage_mv`, `.soc`) + power mode string + TX power

The `fmt_val()` helper already exists (line 985) and properly formats `-0.02` style strings.

### 2. Update `oled_work_handler()` connected phase

Currently at lines 1121-1144, the connected phase switches between IMU and Mic views using `oled_sub_phase` 0/1.

**Change:** Add `oled_sub_phase` 0 = IMU page, 1 = Power page, 2 = Mic PSD page. The auto-rotate logic already exists (time-based dwell). Just need to:
- Increase `oled_sub_phase` range from 0-1 to 0-2
- Add `oled_draw_page_power()` call for sub_phase == 1
- Keep sub_phase == 2 for mic (or shift mic to sub_phase == 2)

### 3. (Optional) Add a scroll indicator

Small dots on the bottom-right (8x16 chars at x=120, y=56) showing current page: `◉◌◌` / `◌◉◌` / `◌◌◉`.

Use chars `●` (0x7F in font) or `°` (0xB0) — though the 8x16 font doesn't include these. Simple workaround: draw a filled 3x3 pixel box for active page, empty 3x3 box for inactive.

## Edge Cases / Risks

| Issue | Mitigation |
|-------|------------|
| Long value strings (e.g. `X-1234.56`) exceed 16 chars | `fmt_val` already caps at `%d.%02u` format, max ~10 chars per value. With label prefix `X` + value + unit, fits in 16 chars |
| Battery voltage shows `0mV` before first read | The `batt_work` starts at `K_NO_WAIT` in `main()`, so first reading arrives within ~70ms. The IMU page runs first, so by the time power page displays, battery data is ready |
| Invert/Contrast settings still apply | `oled_set_pixel` + `oled_flush` are unchanged; settings from OLED_SETTINGS_CHRC still work via `oled_apply_settings()` |
| Deep sleep mode still works | `oled_clear()` + `oled_flush()` are unchanged; the deepsleep path doesn't call `oled_update_connected()` directly |
| Resched. timer for IMU view | Currently uses `imu_dwell_s` setting; with 2 pages the IMU page dwell = `imu_dwell_s/3*2`, power page dwell = `imu_dwell_s/3*1`. Or keep simple: 4s IMU, 3s Power, remaining for Mic |

## Summary

- **No new library** — manual framebuffer stays
- **No new font** — existing 8x16 font used everywhere in connected view (was already used in beacon view)
- **~30 lines of code changed** — just replace `oled_update_connected()` body and add one new draw function
- **Zero RAM increase** — framebuffer is reused, font tables are already loaded
- **Zero flash increase** — font tables are already in the binary
